# 샘플 덤프 데이터

Deep Cognitive Analysis 프롬프트 테스트용 일주일치 더미 데이터

## 파일 구조

- `week-sample.md` - 일주일치 전체 덤프 (Day 1-7)
- `day1-monday.md` - 월요일 덤프 (단일 테스트용)
- `day7-sunday.md` - 일요일 덤프 (단일 테스트용)

## 포함된 패턴

이 덤프 데이터는 다음과 같은 **반복 패턴**을 포함합니다:

### 1. 완벽주의 vs 지속가능성 갈등
- **트리거**: 러닝 루틴, 프로젝트 완성도
- **반응**: "완벽한 루틴"을 지키지 못하면 불안, 찜찜함
- **결과**: 80% 노력의 장기 지속보다 100% 추구 → 번아웃

### 2. 멀티태스킹 vs 집중 문제
- **트리거**: 회고앱, 음성앱 동시 진행
- **반응**: 우선순위를 정해야 한다고 생각하지만 두 개를 왔다갔다
- **결과**: 둘 다 완성 못함, 진전 느림

### 3. 완성 vs 퍼블리싱 갈등
- **트리거**: "완벽하게 만들어야 한다"는 기준
- **반응**: 손을 떼기 어려움, 계속 고치고 다듬음
- **결과**: 퍼블리시 못하고 계속 작업만 함

### 4. 자책 vs 성장 사고방식
- **트리거**: 주간 회고, 스레드 타임라인의 다른 메이커들
- **반응**: "나는 왜 이렇게 느린가?", "시간 낭비했다"
- **결과**: 자책 → 동기 저하 → 더 느려짐

### 5. 인지-행동 불일치
- **인지**: "완벽하지 않아도 괜찮다", "하나씩 집중하라", "퍼블리시하라"
- **행동**: 완벽주의 추구, 멀티태스킹, 완성 못함
- **갭**: "머리로는 아는데 몸과 마음이 따라주지 않음"

## Deep Cognitive Analysis 기대 결과

이 덤프를 분석하면 AI는 다음을 식별해야 합니다:

### 인지 메커니즘
- **멘탈 모델**: "완벽해야 가치있다"
- **가정**: "완벽하지 않으면 실패다"
- **두려움**: "불완전한 것을 보여주면 평가받는다"
- **인과 고리**: 완벽 기준 → 불안 → 과도한 자기 검열 → 퍼블리시 못함

### 레버리지 포인트
- **1순위**: "완벽" 기준을 "충분히 좋음(good enough)"으로 재정의
- **2순위**: 한 프로젝트 완성 후 다음으로 넘어가는 루틴 확립

### 패턴
- 매주 같은 자책 ("이번 주도 실패")
- 매주 같은 계획 오버 (달성률 30% → 다음 주 더 많이 잡음)
- 매번 같은 "알지만 못함" 사이클

### 개선 방향 (고레버리지)
- **왜**: 완벽주의 멘탈 모델이 퍼블리싱을 막고 자책 사이클 만듦
- **무엇을**: "완료 기준"을 명시적으로 정의 (80% 완성 = 퍼블리시)
- **어떻게**:
  1. 회고앱 프롬프트 v3 테스트 → 즉시 퍼블리시 결정
  2. 음성앱은 회고앱 퍼블리시 후로 미루기 (명시적 순서)
- **측정**: 이번 주 내 회고앱 퍼블리시 여부 (Yes/No)
